# StoryBrand Framework Application

**Description:** Positioning brands using narrative structure: hero, guide, problem, solution, transformation.

## The Framework

| Element | Definition |
|---------|-----------|
| **Hero** | The customer — identify who they are and what they want |
| **Problem** | External problem, internal frustration, philosophical wrong |
| **Guide** | The brand — with empathy (I understand) and authority (I can help) |
| **Plan** | Clear, simple steps to engage |
| **Call to Action** | Direct (buy now) and transitional (learn more) |
| **Success** | What life looks like after transformation |
| **Failure** | What's at stake if they don't act |

## Application Process
1. Interview to understand customer's actual language
2. Map external/internal/philosophical problems
3. Position brand as guide, not hero
4. Create one-liner that captures the arc
5. Build all messaging from this foundation

## When to Use
- Any brand strategy project
- Website messaging
- Marketing campaigns
- Product positioning

## When to Avoid
- Technical documentation
- When the 'hero journey' feels forced
- B2B contexts where decision-making is committee-based (adapt carefully)

## Applied In
- Thabiti (traveler as hero, Thabiti as reliable companion)
- Apple Empire (viewer deciding, Aslam sharing genuine experience)

**Success Rate:** 90% — consistently effective across brand projects. Confidence: 93%.
