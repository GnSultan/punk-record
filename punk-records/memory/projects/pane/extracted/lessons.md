# Lessons Learned

*Auto-maintained by Session Observer*

- Research on scroll momentum settling stutter in Electron on macOS ProMotion 120Hz displays revealed multiple root causes: 1) macOS ProMotion VRR drops from 120Hz to 60Hz during momentum scroll deceleration phase as Apple's adaptive refresh rate conserves power when input velocity decreases, 2) Chromium's CVDisplayLink doesn't properly handle VRR transitions causing frame pacing hitches (CADisplayLink fix in progress), 3) Electron's _cornerMask override caused WindowServer GPU spikes on macOS 26 (fixed in Electron 38.2.0+), 4) CSS layout patterns with nested flex + min-height: 0 can create non-fast scrollable regions forcing main-thread scroll fallback
- Electron scroll performance on macOS ProMotion 120Hz: the remaining momentum settling stutter is a known Chromium bug (Issue 40202100) — CVDisplayLink doesn't handle VRR refresh rate tier transitions cleanly during scroll deceleration. This is NOT fixable at the app level. Key fixes that DID help: (1) Remove continuous RAF loops (FPS counter was worst offender), (2) Replace Tailwind animate-pulse with compositor-isolated animation (will-change: opacity + contain: strict), (3) Add overscroll-behavior: contain to scroll containers, (4) Flatten flex nesting depth to reduce non-fast scrollable regions, (5) Remove all custom scrollbar CSS, (6) Remove will-change: transform from scroll containers. Flags that made things WORSE: disable-frame-rate-limit and disable-gpu-vsync dropped FPS to 50.
